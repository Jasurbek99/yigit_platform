YGT Platform — Task System Overview

Tasks drive the 13-step shipment lifecycle. Most are auto-generated by status transitions with rule-based completion; a few are manually created when a user @-mentions an assignee in a comment. Generated 2026-05-09.

Legend — OPEN not started   IN PROGRESS being worked on   DONE completed   BLOCKED waiting on something   CANCELLED dropped  |  auto = system-driven, manual = user-driven

2. Manual tasks (user-created comment-tasks)

Task Trigger to creation OPEN → IN PROGRESS IN PROGRESS → DONE
Comment-task
ShipmentComment with assignee		 User posts a comment via POST /api/v1/export/comments/ with an assignee (typically by typing @user:N or @role:warehouse_chief in the Comments Drawer). Only root comments can be tasks; replies cannot. Implicit — the comment-task lives in is_done=false until marked done. There is no separate "in progress" state on comment-tasks. Assignee or comment author calls POST /api/v1/export/comments/{id}/done/ → sets is_done=true, done_at=now(), done_by=user. Reopen: POST /comments/{id}/reopen/.

3. State machine (rule-driven Task model)

From To How Endpoint / Code
— OPEN Shipment status transition fires generate_tasks_for_status(). Idempotent — won't re-create existing (shipment, rule) pairs. services/shipment.py · services/task_rules.py
OPEN IN PROGRESS Auto: first edit to a target field on the shipment fires /start/ from the My Task card (debounced).
Manual: user clicks the Start button. 		POST /api/v1/export/tasks/{id}/start/
IN PROGRESS DONE Field-driven (auto): resolve_for_shipment() runs on every Shipment.save(). If the rule's completion_rule is ALL_FIELDS_FILLED or ANY_FIELD_FILLED and the target_fields are satisfied, the task is closed: state=DONE, completed_at=now().
Manual: for MANUAL_DONE rules, user clicks "Mark Done" on the My Task card. 		POST /api/v1/export/tasks/{id}/complete/
services/task_rules.py:313–356
OPEN / IN PROGRESS BLOCKED User clicks "Block" with a reason; optionally links blocked-by tasks (M2M). POST /api/v1/export/tasks/{id}/block/
BLOCKED IN PROGRESS User clicks "Unblock" from the Other Tasks row modal. POST /api/v1/export/tasks/{id}/unblock/
any CANCELLED Admin or director cancels (e.g. is_gapy_satys flipped, rule no longer applies). POST /api/v1/export/tasks/{id}/cancel/

4. Where tasks surface in the UI

SurfaceWhat it shows"Today" filter
My Tasks page /me/tasks/ All tasks where assignee_role == user.role. Filterable by state, step, overdue=true. Tasks with completed_at on today's date (state=DONE) appear under the "Done" filter; the overdue filter highlights anything past its deadline.
Shipment Detail — My Task card The single active task (OPEN / IN PROGRESS / BLOCKED) for the current user on this shipment, with progress bar from target_fields_list. "Mark Done" button only for manual_done rules. n/a — always shows the live task.
Shipment Detail — Other Tasks row All other tasks on the shipment (including DONE / CANCELLED). Clicking a row navigates to its target fields. BLOCKED rows open a modal with the blocked reason and an Unblock button. Closed-today tasks are visually distinct (DONE pill + completed_at timestamp).
Sheet view toolbar Shows task_counts per shipment from the wrapped /sheet/ response: {open, done, assigned_to_me_open}. Indicator badge for "open tasks assigned to me" lights up in real time.
